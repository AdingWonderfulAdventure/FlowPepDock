# 8. 局限性与下一步

> 文档类型：当前代码说明
>
> 本文只总结当前主线仍然成立的局限性和后续方向，不展开历史实验过程。

## 8.1 当前已知局限性

### 1. 最佳 checkpoint 仍来自 NPZ 通路

虽然 `shard` 通路更快，但当前最佳点仍来自 `npz graph_main`。

### 2. shard 训练当前略低于最佳 npz

`shard` 已经稳定、快速，但主指标通常略低于最佳 `npz graph_main`。

### 3. 图模型多卡还未切到 DDP

当前图模型没有正式 DDP 版本。

原因：

- 原生 `DataParallel` 会切坏 `edge_index`

### 4. 当前还没有引入 ESM

这不是当前缺陷，而是刻意延后。

当前判断是：

- 现阶段没有必要为了主线补 ESM

### 5. 当前轻量化收益主要不在删特征

当前实验已经说明：

- 真正速度瓶颈不在少数特征维度
- 更可能在 IO / batch / graph forward

## 8.2 当前最合理的模型侧下一步

当前不建议再继续大改模型结构。

若继续做模型研究，最合理的是：

- 保持 `graph_main` 作为主线
- 保持当前 `graph_main` 主线，不在现阶段恢复历史轻量化归档分支
- 不上 ESM
- 不上 atom-level
- 不上双尺度图

## 8.3 当前最合理的工程侧下一步

如果目标是“更快”，下一步优先级应转向：

- shard 读取继续优化
- dataloader / batch cache
- graph forward profiling
- DDP 多卡训练

## 8.4 当前最合理的交付侧下一步

如果目标是毕业设计/论文/汇报，下一步更值得做：

- 方法摘要图
- 实验总表
- 消融表
- 写作材料整理

## 8.5 当前关于 ESM 的正式建议

当前不建议把 ESM 加到主线。

理由：

- 当前主线已经成立
- 当前短板不是序列语义
- 当前更重要的是交付与工程化

若未来做 `v2`，ESM 可以作为增强版分支考虑。
